üåê What is Web Development?

Frontend Development

The client-side of web development. What users see and interact with directly.

  • HTML (Structure)
  • CSS (Styling)

The server-side of web development. Handles data, logic, and server operations.

  • APIs & Web Services

Combining both frontend and backend development skills.

  • Complete Web Applications
  • End-to-End Development
  • DevOps & Deployment
  • Performance Optimization

üìù HTML Basics

HTML (HyperText Markup Language) is the foundation of every website. It provides the structure and content.

Essential HTML Tags

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>
<body>
    <header>
        <h1>Main Heading</h1>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#about">About</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <section>
            <h2>Section Heading</h2>
            <p>Paragraph text here.</p>
        </section>
    </main>
    
    <footer>
        <p>© 2025 Your Website</p>
    </footer>
</body>
</html>

Common HTML Elements

Text Elements

  • <h1> - <h6> - Headings

  • <span> - Inline text
  • <strong> - Bold text
  • <em> - Italic text

Structure Elements

  • <div> - Container
  • <section> - Content section
  • <article> - Article content
  • <header> - Page/section header
  • <footer> - Page/section footer

Interactive Elements

  • <a> - Links
  • <button> - Buttons
  • <input> - Form inputs
  • <form> - Forms
  • <img> - Images

CSS (Cascading Style Sheets) controls the visual presentation of your HTML content.

CSS Syntax & Selectors

/* Element Selector */
p {
    color: #333;
    font-size: 16px;
}

/* Class Selector */
.container {
    max-width: 1200px;
    margin: 0 auto;
}

/* ID Selector */
#header {
    background-color: #007bff;
    padding: 20px;
}

/* Attribute Selector */
input[type="text"] {
    border: 1px solid #ccc;
    padding: 10px;
}

üì± Responsive Design

Make your websites look great on all devices and screen sizes.

Mobile-First Approach

/* Mobile styles (base) */
.container {
    padding: 10px;
    font-size: 14px;
}

/* Tablet styles */
@media (min-width: 768px) {
    .container {
        padding: 20px;
        font-size: 16px;
    }
}

/* Desktop styles */
@media (min-width: 1024px) {
    .container {
        padding: 30px;
        font-size: 18px;
        max-width: 1200px;
        margin: 0 auto;
    }
}

Responsive Units

Relative Units

  • em - Relative to parent element
  • rem - Relative to root element
  • % - Percentage of parent
  • vw/vh - Viewport width/height

Absolute Units

  • px - Pixels (avoid for responsive)
  • pt - Points (print media)
  • cm/mm - Centimeters/millimeters
  • in - Inches

‚úÖ Best Practices

Follow these guidelines to write clean, maintainable, and professional code.

üîç SEO Optimization

  • Use semantic HTML5 elements
  • Add proper meta tags
  • Optimize images with alt text
  • Create clean, descriptive URLs
  • Use heading hierarchy (h1-h6)

‚ôø Accessibility

  • Use proper ARIA labels
  • Ensure keyboard navigation
  • Maintain color contrast ratios
  • Provide alternative text for images
  • Use focus indicators

‚ö° Performance

  • Optimize images and assets
  • Minimize HTTP requests
  • Use CSS and JS compression
  • Implement lazy loading
  • Cache static resources

üßπ Code Quality

  • Use consistent naming conventions
  • Comment your code appropriately
  • Follow DRY principle
  • Keep functions small and focused
  • Use version control (Git)
